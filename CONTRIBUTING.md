# Contributing

Thanks for your interest in improving the ExecutionRank whitepaper.

## What contributions are welcome
- Typos, clarity, and structure edits
- Improved notation consistency
- Additional citations / related work (with BibTeX entries)
- Threat-model / security analysis improvements
- Reproducible evaluation suggestions

## How to contribute
1. Fork the repo
2. Create a branch: `git checkout -b fix/short-description`
3. Make changes under `paper/`
4. Ensure the paper still compiles locally (see repo README)
5. Open a PR with a concise description of the change

## Style
- Keep tone technical (no marketing language)
- Prefer precise definitions over prose
- Keep assumptions explicit
